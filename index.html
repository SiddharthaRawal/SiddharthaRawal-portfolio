About Me

Understanding the journey behind the code

01

Why Computer Science

I became interested in computer science because it provides a formal framework for solving problems systematically. What drew me in was not the technology itself, but the underlying process: breaking complex systems into smaller, testable parts, understanding constraints, and iterating toward solutions that work. I find value in the discipline required to write code that is correct, readable, and maintainable. These principles‚Äîclarity, correctness, and simplicity‚Äîextend beyond programming and shape how I approach learning in general.

02

How I Learn

I learn by building. When I encounter a concept I don't understand, I implement it from scratch, break it, and fix it until the behavior makes sense. This process is slow, but it produces durable understanding. I prefer depth over breadth‚Äîspending time on fundamentals rather than rushing through surface-level tutorials. My projects reflect this approach. They are not polished products, but exercises in understanding: how does a data structure perform under different conditions? How do I optimize an algorithm without sacrificing readability?

03

What I'm Working On

Right now, I'm focused on strengthening my foundations in algorithms, data structures, and systems-level thinking. I'm working through problems that require careful analysis of time and space complexity, and I'm building small projects that force me to think about memory management, concurrency, and trade-offs between different approaches. I'm also learning to read and understand existing codebases‚Äînot just to copy patterns, but to see how experienced engineers make architectural decisions. This skill feels as important as writing code itself.

04

Long-term Perspective

I see computer science as a decades-long learning path, not a race to acquire skills for immediate use. I'm more interested in building the capacity to reason clearly about hard problems than in chasing trends or optimizing for short-term outcomes. I want to study computer science at a rigorous institution where I can learn from faculty who care about first principles, work alongside peers who value depth, and develop the discipline required to contribute meaningfully to the field over time.

Plagiarism Detection System

A text similarity detection system that analyzes documents to identify potential plagiarism using string matching algorithms and statistical analysis. Built to understand how similarity can be quantified programmatically.

Python Natural Language Processing String Algorithms File I/O
Click to view full size

Challenge

Accurately detecting similarity while avoiding false positives from common phrases, proper citations, and paraphrased content. The system needed to distinguish between coincidental similarity and deliberate copying.

Solution

Implemented multiple comparison algorithms including Levenshtein distance for character-level analysis and cosine similarity for semantic matching. Used weighted scoring based on phrase length, uniqueness, and structural patterns to provide nuanced similarity metrics.

Learning

Gained practical experience with string algorithms and learned the importance of balancing precision with performance. Discovered that text analysis requires understanding context, not just pattern matching‚Äîa reminder that good algorithms need good heuristics.
